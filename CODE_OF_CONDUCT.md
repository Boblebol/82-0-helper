# Code Of Conduct

This project should stay practical, respectful, and welcoming.

## Expected Behavior

- Be clear and constructive.
- Assume good intent when discussing bugs or design tradeoffs.
- Keep feedback focused on the code, data, documentation, and user experience.
- Respect that maintainers and contributors may have different levels of time and context.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Repeated off-topic comments that make project work harder.
- Spam or low-effort automation that creates maintenance burden.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or contributors that violate this code of conduct.
